# iOS-App-test
Начало выполнения 19.02.2023, окончание 24.02.2023
##  Выполненные задачи:
Выполнены все задачи, указанные в тестовом задании: 
- Хранение всех данных локально с помощью базы данных CoreData, UserDefaults, статически в зависимости от функционала и цели
- Все экраны реализованы 
- Обработаны все ошибки
- Есть валидация данных
- Большинство элементов пользовательского интерфейса реализованы с помощью Storyboad и Auotolayout, а контроллер NewsController и ячекйка NewsCell сверстаны программно, используя SnapKit
- Новости создаются с помощью массива данных, которые написаны вручную и хранятся в CoreData
- При повторном включении приложения, данные отображаются из базы данных, а не из массива, описанный в предыдущем пункте
- При каждом запуске проверяется массив новостей и сравнивается с базой данных по id, если такой новости нет в списке, то новость добавляется, в противном случае ничего не происходит
- При нажатии на лайк (избранное), состояние меняется на противоположное на всех экранах, также все меняется при нажатии из любого экрана на лайк и состояние сохранятется в базе данных и добавляется на экран "избранное"
- "Избранное" берет данные из базы данных, фильтрует данные по состоянию (isFavorites) и передает полученный массив в источник данных
и т.д.
